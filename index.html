Anatolia MUD Visual Area Editor

An easy to use GUI App for editing area files.

What is MUD?

Multi-User Dungeon (MUD) is a computer-based text game which several players play at the same time, interacting with each other as well as with characters controlled by the computer.

What is Anatolia MUD Editor?

Anatolia MUD Editor is a a GUI editor for the area files of a Multi-User Dungeon (MUD) game called AnatoliaMUD.

About Our Project

Our aim was to create a modern and user-friendly graphical editor specifically designed for AnatoliaMUD to make creating, browsing and updating area files as easy and accessible as possible.

Problem Definition

  • Our problem is how a GUI program should be programmed and designed to allow for the editing of MUD area files, which have unique and hard to grasp layouts or object relations; in an easy to grasp, intuitive and comprehensive way

  • An area file has 11 sections which represent the various entities and features within an area such as rooms or mobiles; however these are not implemented in an object-oriented way where each instance of a mob is declared: A feature called resets handles almost everything related to “instance” management and initialization of these entities. This makes it very hard to read as a result, and can cause increased amounts of human error.

  • In order to create an intuitive GUI editing system for such files, all rooms and entities in them have to be shown in an “instance” based hierarchy where the user can clearly see which room contains which entities.

Proposed Solution

Our proposed solution for this problem is to create an IDE-like native interface application where the user can see and edit the hierarchical layout of the area file, and edit any specific elements within this hierarchy such as rooms, mobs, objects and all the variables within these elements through the GUI.

Methodology

  1. Understanding of .are files, how to read them/edit them.

  2. Implement the parser and convert text fields to java objects and variables.

  3. Design a GUI interface for easy user interaction.

  4. Building the connections between parser and GUI interface.

Project Details

The Anatolia MUD Editor consists of multiple packages and classes organized in accordance to the package by feature paradigm. There are 5 main packages in Anatolia MUD Editor.

  • com.amve.area: This package consist of classes that stores parsed data.

  • com.amve.globals: This package consist of Global Variables and Enum Tables.

  • com.amve.gui: This package has the Main GUI component.

  • com.amve.parser: Parser package is responsible for reading the area files and structuring them.

  • com.amve.utils: Various helper/utility classes.

GUI Design

The GUI was designed with simplicity and familiarity in mind, achieved through native OS application styling and common IDE/editor UI parts such as SashForms.

The Results

  • There are no others MUD area editors that work with our AnatoliaMUD version.

  • With the use of our editor, the AnatoliaMUD community can contribute to the editing and creation of new area files with the help of an easy to use and intuitive GUI.

  • New additions such as custom maps could possibly contribute to a wider audience.

  • This project could be improved by adding other viewing/data altering methods to itself:

    • A graph that shows connections between rooms.

    • Drag and drop altering of objects.

    etc.

  • There are several shortcomings about this project:

    • Original area file order is not preserved.

    • Parser implementation can return incorrect values.

    • Not every flag has been properly mapped to checkboxes/buttons.

    These must be rectified in order to bring the product to a higher level of completeness.
